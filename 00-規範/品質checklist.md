# 品質 checklist

每次**新建**或**重大更新**卡片 wiki 後，跑這份 checklist 自我檢查。一張卡通過所有檢查才算「發布」。

## A. YAML frontmatter

- [ ] `type` 正確（`card` / `card_index` / `card_alias`）
- [ ] `bank` 與檔名前綴一致（檔名 `凱基-誠品聯名卡.md` → `bank: 凱基銀行`）
- [ ] `issuer` 為**法定全名**（如「凱基商業銀行」而非「凱基」）
- [ ] `network` 列出所有版本（不只主版本）
- [ ] `annual_fee` 完整描述（金額 + 首年規則 + 達標免年費條件）
- [ ] `status` 在 enum 內（見 [[新增卡片wiki範本|新增卡片 wiki 範本]] § status enum）
- [ ] `review_on` 已設定，未過度集中（見 [[複查觸發點]]）
- [ ] `aliases` 至少列出 3 個（含活動名稱、簡稱、英文名）

## B. 內容完整性

- [ ] `## 定位`：1-2 段，含差異化亮點與推薦判斷核心
- [ ] `## 目前可用回饋`：**表格**形式，5 欄齊全（情境 / 回饋 / 上限 / 有效期間 / 來源）
- [ ] 每筆回饋條款都有對應 `## 使用條件` 說明（若條件複雜）
- [ ] `## 公開銀行關係`：含 wikilink 到銀行頁 + 共用額度宣告
- [ ] `## 排除條款`：至少 5 項（分期、稅費、預借、第三方支付、海外實體 vs 網購區分）
- [ ] `## 適合情境` 與 `## 不適合情境` 各 3-5 點
- [ ] `## 來源`：至少 1 個官方頁 URL + 對應 `10-來源/` 筆記
- [ ] `## 更新紀錄`：本次變更日期 + 動作描述

## C. 官方 vs 第三方

- [ ] 所有 headline 回饋率（如「8% 回饋」「海外 3.5%」）**都引官方頁**
- [ ] 第三方榜單（卡優、CreditCards、袋鼠金融、Money101）**只列為訊號來源**，不單獨支撐回饋率
- [ ] 若官方與第三方數字不一致 → 以官方為準，於更新紀錄註記差異
- [ ] 第三方提及的「最高 X%」**疊加邏輯**已用官方頁拆解（基本 + 加碼 + 通路 + 登錄）

## D. 公開 / 私有分層

- [ ] **不包含**個人信用額度、調額紀錄、申辦結果、客服對話、簽帳資料
- [ ] **不包含**個人持卡狀態（即使是我自己的，公開頁也不寫）
- [ ] **不包含**完整卡號、身分證字號、帳號、客服驗證資訊
- [ ] 銀行共用額度為**規則**（可寫），個人實際額度**不可寫**
- [ ] 若需保存個人狀態，存於本地不入 git 的私有區（依 [[00-規範/公開私有分層|公開私有分層]]）

## E. wikilink 與導航

- [ ] `## 公開銀行關係` 連到 `[[20-Wiki/銀行/{銀行}]]`
- [ ] 通路 / 策略相關處連到對應 `[[20-Wiki/通路/X]]` 或 `[[20-Wiki/策略/X]]`
- [ ] `## 來源` 區連到 `[[10-來源/...]]` 對應筆記
- [ ] 跨銀行比較表中對應卡別連到對應 wiki（如有）
- [ ] 別名頁正確連到 `canonical` 主檔

## F. 命名與結構

- [ ] 檔名遵循 `銀行-卡名.md` 命名規則（見 [[新增卡片wiki範本]]）
- [ ] 區段順序遵循模板（定位 → 回饋 → 使用條件 → 公開銀行關係 → 排除條款 → 適合 / 不適合 → 跨銀行比較 → 來源 → 更新紀錄）
- [ ] 標題層級正確（卡片名為 `#`，主區段為 `##`，子區段為 `###`）
- [ ] 中文標點正確（不用半形逗號、空格習慣統一）

## G. 跨頁一致性

- [ ] `30-研究/銀行/{銀行}.md` 已更新「已整理卡片」表格
- [ ] `index.md` 已加入新卡的 wikilink
- [ ] 若新增策略 / 通路頁 → `index.md` 與相關策略頁同步更新
- [ ] 該月 `log/YYYY-MM.md` 已加入本批次紀錄

## H. 風險與排除條款

- [ ] 預借現金 / 特約分期是否回饋已明確寫
- [ ] 稅款 / 規費 / 保費是否回饋已明確寫（保費若特殊規則需單獨列）
- [ ] 第三方支付（LINE Pay / 街口 / Apple Pay / 等）的認定方式已寫
- [ ] MCC / 行業代碼 / 商店名稱認定限制已寫（若有）
- [ ] 海外實體 vs 海外網購區分已寫（若有不同）
- [ ] 歐洲指定國家排除（若有）已寫
- [ ] 退貨會扣回回饋已寫（若銀行有此規則）

## I. 推薦判斷品質

- [ ] 「適合情境」用**消費結構**描述（如「月行動支付 ≥ NT$5,000」），不用「常用 LINE Pay」之類模糊描述
- [ ] 「不適合情境」明確指出**何時不該辦**這張卡
- [ ] 跨銀行比較表至少 3 行（除非該卡無可比對象）
- [ ] 高回饋宣稱（≥ 5%）有對應上限說明，避免誤導
- [ ] 新戶限定優惠與既有卡友權益**明確區分**

## 發布門檻

| 卡片類型 | 必須通過 | 可豁免 |
|---|---|---|
| 標準現金回饋 / 哩程卡 | A / B / C / D / E / F / G / H | I（推薦判斷可後續加） |
| 頂級卡 / 邀請制 | A / B / D / E / F / G | C 的「官方頁完整」可豁免（資訊本身難取得） |
| 卡群索引 | A / B（簡化）/ E / F / G | C / D / H（這些屬於每張卡頁的責任） |
| 別名指向頁 | A / E | 其他都豁免 |
| 停辦索引 | A / B（簡化）/ E / F | C / D / H |
| Skinny 初版 | A 部分 + B 部分 + E + F | 其他標待補核 |
