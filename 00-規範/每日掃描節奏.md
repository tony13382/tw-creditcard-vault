# 每日掃描節奏

這份 SOP 規定如何在日常維護這個 vault：哪些動作要每天做、哪些等到每週、哪些等到每季。**目的不是要每天花很多時間**，而是讓累積的工作可控、不會集中在 H1/H2 邊界爆炸。

## 三層節奏

### Layer 1：每日 5-10 分鐘（輕量擷取）

執行時機：早上開信箱 / 看新聞時順手做。

| 動作 | 工具 | 結果 |
|---|---|---|
| 開啟 `觀察清單.md` 掃一遍 | Obsidian | 心中有當前未決事項 |
| 看到新活動 / 新卡 / 銀行公告 → 擷取**連結 + 一句話** | 任何記事 | 進入 `10-來源/` inbox（檔名 `YYYY-MM-DD-銀行-關鍵字.md`）|
| **不寫深度內容**，不查官方頁，不做交叉比對 | — | 等每週再升級 |

**原則**：每日只擷取，不分析。分析的時間在每週。

### Layer 2：每週 30-60 分鐘（維護）

執行時機：每週固定一天（建議週末或週一）。

#### 例行檢查

```
Grep 工具：review_on: 2026-MM-DD（本週 + 下週日期）
→ 列出本週到期卡片
```

逐張卡片：
1. 開啟卡片 wiki
2. 連到 `來源` 區的官方頁
3. 確認回饋條款是否仍有效
4. 若有變動 → 更新「目前可用回饋」表 + 加入「更新紀錄」一行
5. 推遲 `review_on` 至下次合理日期（見 [[複查觸發點]]）

#### Inbox 升級

對該週擷取進 `10-來源/` 的線索，逐一決定：
- **升級 wiki**：資料夠 → 跑 [[新增卡片 wiki 範本]] 建卡片頁
- **留 backlog**：資料不足 → 補進 `30-研究/` 對應銀行的研究頁的「待研究方向」
- **丟棄**：過期 / 第三方謠言 / 與既有 wiki 重複

#### 新銀行 / 新卡上市

- 該週若有新銀行進入金管會清單 → 在 `30-研究/銀行/` 建研究頁，狀態 `research_pending`
- 該週若銀行發新卡 → 建來源筆記 + 建初版 wiki（可採 `skinny` 狀態）

### Layer 3：每季 / H1-H2 邊界（大盤）

執行時機：1月、4月、7月、10月各一次（每季首週末），其中 1月 / 7月 為 H1/H2 邊界，工作量最大。

#### 必跑動作

1. **發卡機構清單對齊**：跑金管會最新「信用卡重要業務及財務資訊揭露」（每月公布），更新 `30-研究/發卡機構清單.md`
2. **產品總覽 diff**：對前 10 家熱門銀行（占榜單 80%）跑官方信用卡總覽 vs 現有 wiki
   - 新增卡 → 建研究 / wiki
   - 停發卡 → 卡片狀態改為 `stopped_issuance_existing_only`，加入該銀行「停辦卡群索引」
3. **跨銀行比較頁更新**：對主要主題（海外無上限、行動支付、JCB、加油、寵物、頂級卡）跑 `20-Wiki/策略/`
4. **過期描述掃描**：Grep `2026 H1`、`至 2026-06-30`、`新戶 90 天內` 等過期描述

#### H1/H2 邊界額外動作（1月 / 7月）

- 過去半年公告全部複查（即使 `review_on` 未到）
- 月 log（`log/YYYY-MM.md`）做半年回顧

## 觸發式（非排程）

下列事件**立即**觸發處理，不等下次例行：

| 事件 | 動作 | 時限 |
|---|---|---|
| 新卡上市新聞 | 建來源筆記 + 初版 wiki | 7 天內 |
| 銀行宣布停發某卡 | 卡片狀態改為 `stopped_issuance_existing_only`、加入停辦索引 | 24 小時內 |
| 年費政策變動 | 卡片頁 `annual_fee` 更新 + 更新紀錄 | 24 小時內 |
| 銀行併購（如花旗 → 星展）| 建「業務轉移」索引頁 + 既有卡換卡建議 | 7 天內 |
| 金管會卡數異動公告 | 更新 `30-研究/發卡機構清單.md` | 該月內 |

## 自動化提示（可直接交給 Claude 跑）

下列任務適合每週 / 每季交給 Claude，提示模板：

### 每週到期複查
```
跑 Grep 在 20-Wiki/信用卡/ 找出所有 review_on: <本週至下週日期> 的卡片，
列出檔名 + 卡片名 + 當前 status；對每張卡建議下一步動作。
```

### 銀行 diff
```
對 X 銀行：抓取官方信用卡總覽頁，盤查所有現行個人卡（不含法人卡、
不含 debit card），跟 20-Wiki/信用卡/X-* 對照，列出：
（a）官方有但 vault 沒有的卡 → 需要建 wiki
（b）vault 有但官方查無的卡 → 可能已停發
（c）vault 與官方資料差異
```

### 過期掃描
```
跑 Grep 找出所有 wiki 含「2026 H1」「至 2026-06-30」「2026/1/1 至 2026/6/30」
等已過期描述，列出檔名與行號。
```

### 跨銀行比較表更新
```
從 20-Wiki/信用卡/ 找出所有 海外回饋 ≥ 2% 的卡片，列出：
銀行、卡別、海外回饋率、月上限、需登錄條件、年費。
產出新的「海外無上限卡」比較表。
```

## 工作量預估

| 節奏 | 平均時間 | 高峰時間（H1/H2 邊界）|
|---|---|---|
| 每日輕量擷取 | 5-10 分鐘 | 不變 |
| 每週維護 | 30-60 分鐘 | 60-90 分鐘 |
| 每季大盤 | 半天 | 1-2 天（1月 / 7月）|

預估每月固定工時：每日 5 分 × 30 + 每週 45 分 × 4 = **約 5.5 小時 / 月**。每季再加 0.5-2 天。

## 不要做的事

- **不要**等到資料 100% 完整才寫 wiki：可用 `skinny` 狀態先放，標待補核
- **不要**把第三方榜單數字當官方資料：必須引官方頁 URL 為來源
- **不要**個人持卡狀態 / 信用額度寫入公開 wiki（見 [[00-規範/公開私有分層|公開私有分層]]）
- **不要**重複跑同一銀行大盤：已建 wiki 用 H1/H2 補核更新即可
- **不要**讓 `review_on` 過度集中在同一天：見 [[複查觸發點]] 的散點配置原則
