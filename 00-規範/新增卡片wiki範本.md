# 新增卡片 wiki 範本

這份 SOP 規定卡片 wiki 的標準結構、必填欄位與命名規則。每次新增 `20-Wiki/信用卡/` 內的檔案都應遵循本範本。

## 命名規則

| 檔案類型 | 命名格式 | 範例 |
|---|---|---|
| 一般卡 | `銀行-卡名.md` | `凱基-誠品聯名卡.md` |
| 卡群索引 | `銀行-卡群名群.md` | `第一銀行-商旅卡群.md` |
| 別名指向頁 | `銀行-別名.md` → `canonical` 連到主檔 | `陽信-曜晶卡.md` → 主檔 `陽信-VISA曜晶卡.md` |
| 停辦索引 | `銀行-停辦卡群.md` | `台灣樂天-停辦卡群.md` |

**命名注意**：
- 不要用空格，用 `-` 分隔
- 銀行名稱用最常見的短名（凱基、第一銀行、永豐 等）
- 不要中英混雜（如 `Card` 改 `卡`）
- 同卡不同版本（如悠遊卡 / 一卡通）優先合併單頁，差異於頁內表格區分
- 同銀行 ≥ 5 張低密度卡 → 建索引頁，不要每張都單獨建檔

## YAML frontmatter 必填欄位

### 一般卡（`type: card`）

```yaml
---
type: card
bank: 凱基銀行
issuer: 凱基商業銀行  # 法定全名
network:  # 列出所有版本
  - Visa Signature
  - Mastercard Titanium
shared_limit_bank: 凱基銀行  # 共用額度的銀行
annual_fee: 主卡 NT$1,200；附卡免；首年免；達標可免次年
status: active_with_conditions  # 見下方 status enum
review_on: 2026-12-15  # 下次複查日期
aliases:  # 所有別名（含活動名稱、簡稱、英文名）
  - 凱基誠品聯名卡
  - 凱基 誠品聯名卡
  - 誠品聯名卡
  - KGI Eslite Co-brand Card
---
```

### 卡群索引（`type: card_index`）

```yaml
---
type: card_index
bank: 第一銀行
issuer: 第一商業銀行
network:
  - Mastercard Titanium
  - Visa Signature
shared_limit_bank: 第一商業銀行
annual_fee: 依各卡公告
status: active_with_conditions
review_on: 2026-12-15
aliases:
  - 第一銀行商旅卡群
  - 鈦金商旅卡
  - 雙幣鈦金商旅卡
  - 御璽商旅卡
  - 白金商旅卡
---
```

### 別名指向頁（`type: card_alias`）

```yaml
---
type: card_alias
bank: 陽信銀行
canonical: 陽信-VISA曜晶卡  # 指向主檔（不含 .md）
status: alias_redirect
review_on: 2026-12-15
aliases:
  - 陽信曜晶卡
  - 曜晶卡
---
```

別名頁內容應極簡：標題 + 一段「請參閱 ➡️ [[主檔]]」+ 1-2 段補充說明 + 更新紀錄。

## status enum

| status | 含意 | 適用情境 |
|---|---|---|
| `active_with_conditions` | 在售，有特殊條件 / 加碼 | 90%+ 卡片 |
| `active_existing_customers_only` | 在售，但只限既有客戶 | 邀請制變相停發 |
| `invitation_only` | 邀請制 | 高階卡如凱基無限卡 |
| `skinny_pending_*` | 資料不完整 | 初版 wiki（標明哪邊待補）|
| `stopped_issuance_existing_only` | 停止發行，既有卡友可繼續用 | 樂天玫瑰金卡、富邦悍將卡 |
| `merged_into_*` | 已整併入其他卡 | 台新玫瑰太陽卡 → Richart |
| `business_transferred` | 業務已轉移到其他發卡機構 | 花旗 → 星展 |
| `active_low_priority` | 在售但低優先 | 其他卡群索引、不適合主力推薦的卡 |
| `alias_redirect` | 別名指向頁 | 同卡多名稱整理 |

## 必填區段（依序）

### 1. `# 銀行卡名`（標題）

```markdown
# 凱基誠品聯名卡
```

### 2. `## 定位`

1-2 段，說明：
- 這張卡是什麼類型（現金回饋 / 哩程 / 認同卡 / 頂級）
- 主要差異化亮點（1 句話）
- 推薦判斷核心（哪些條件決定值不值得辦）

### 3. `## 目前可用回饋`

**必須**用表格呈現，5 欄：

```markdown
| 情境 | 回饋 | 上限 | 有效期間 | 來源 |
|---|---:|---:|---|---|
| 一般消費 | 0.7% 現金回饋 | 無上限 | 持續 | [[來源筆記]] |
| 誠品 | 黑卡 7% / 御璽卡 3% | 每月帳單 NT$300 | 2026-01-01 至 2026-06-30 | 同上 |
```

**所有回饋率必須引官方頁**。第三方榜單只能在「定位」中提及，不可放入回饋表。

### 4. `## 使用條件`（可選，但回饋複雜時必填）

說明觸發每筆加碼的具體條件（單筆金額、登錄需求、帳單方式、自扣戶、新戶 / 既有卡友、活動期間等）。

### 5. `## 公開銀行關係`

```markdown
## 公開銀行關係

- 所屬銀行：[[20-Wiki/銀行/凱基銀行|凱基銀行]]
- 是否共用銀行額度：是
- 個人信用額度、持卡狀態、調額紀錄：不納入公開卡片頁。
```

### 6. `## 排除條款`

至少列出：
- 預借現金 / 特約分期是否回饋
- 稅款、規費、保費是否回饋（保費另有特殊規則時要寫清楚）
- 第三方支付排除規則
- 海外網購 vs 海外實體區分（如有）
- 行業代碼 / MCC 認定限制

### 7. `## 適合情境` + `## 不適合情境`

各 3-5 點，**用消費結構描述**（如「月行動支付 ≥ NT$5,000」），不用個人偏好。

### 8. `## 跨銀行比較`（可選但建議）

對主要競爭對手做表格比較。**只有當該卡有跨銀行可比的主題時建**（海外無上限、加油、JCB、寵物、頂級等）。

### 9. `## 來源`

列出**所有**官方產品頁、活動頁、PDF 條款的 URL + 對應的 `10-來源/` wiki 連結。

第三方榜單（CreditCards、卡優、袋鼠金融、Money101 等）可作為**訊號來源**列出，但**不可單獨支撐回饋率宣稱**。

### 10. `## 更新紀錄`

每次重大變更（不只 typo）寫一行：

```markdown
- YYYY-MM-DD：[動作描述]，記錄 [主要回饋]、[年費政策]、[卡別]。
- YYYY-MM-DD：YYYY H[1/2] 補核。[H2 公告 / 未公告] [變更 / 延續]。複查日期推遲至 YYYY-MM-DD。
```

## 區段排序（嚴格依照）

```
1. 定位
2. 目前可用回饋
3. [可選] 點數機制 / 使用條件 / 額外權益
4. 公開銀行關係
5. 排除條款
6. 適合情境
7. 不適合情境
8. [可選] 跨銀行比較
9. 來源
10. 更新紀錄
```

## 建立前檢查（避免重複建檔）

每次建立新卡 wiki 前**必跑**：

1. **Glob `*別名*`** 確認檔案是否已存在
2. **Grep `aliases:` + 卡名**確認別名是否被其他檔案宣告
3. 若已存在 → 更新既有檔案，不要建新檔
4. 若同卡有不同版本（悠遊 / 一卡通）→ 優先合併單頁
