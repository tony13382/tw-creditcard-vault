# 熱門信用卡資料抓取需求

## 目的

這份需求用來把研究節奏從「逐銀行整理」擴充成「熱門候選池」。熱門度只決定先研究哪張卡，不代表正式推薦。

## 資料來源分層

| 層級 | 來源 | 用途 | 可否直接推薦 |
|---|---|---|---|
| H0 官方條款 | 銀行產品頁、活動頁、注意事項、PDF 條款 | 正式回饋、上限、排除條款、活動期間 | 可，但需查核完整 |
| H1 官方市場統計 | 金管會信用卡重要業務與財務資訊 | 發卡機構層級動能：有效卡、發卡數、簽帳金額 | 不可，僅作銀行動能 |
| H2 第三方榜單 | Roo.Cash、iCard.AI、Money101 等 | 熱門候選與研究優先級 | 不可，僅作線索 |
| H3 社群/搜尋 | PTT、Dcard、Google Trends、新聞 | 討論熱度、疑點與使用者問題 | 不可，僅作線索 |
| H4 個人化資料 | 個人持卡、額度、帳戶、會員等級、消費偏好 | 私有推薦套用 | 不進公開 repository |

## 抓取欄位

| 欄位 | 說明 | 必要性 |
|---|---|---|
| 卡片名稱 | 第三方榜單顯示名稱與官方卡名 | 必填 |
| 發卡機構 | 銀行或信用卡公司 | 必填 |
| 熱門來源 | Roo.Cash、iCard.AI、Money101、社群、官方活動等 | 必填 |
| 榜單類型 | 即時熱門、本月熱門、主題推薦、十大必辦、文章推薦 | 必填 |
| 榜單日期 | 擷取日期與頁面標示日期 | 必填 |
| 榜單位置 | 若可辨識，記錄排名或大致順序 | 建議 |
| 回饋 headline | 第三方顯示的最高回饋或主打情境 | 建議 |
| 官方產品頁 | 後續官方查核用 | 必填，若未查先留空 |
| 官方活動頁 | 後續官方查核用 | 必填，若未查先留空 |
| 既有 wiki | 是否已整理進 `20-Wiki/信用卡/` | 必填 |
| 熱門度分數 | 依 [[20-Wiki/規則/信用卡熱門度評分模型|信用卡熱門度評分模型]] | 必填 |
| 研究狀態 | 候選、待官方查核、已建 wiki、需修復、暫緩 | 必填 |
| 下一步 | 官方查核、補條款、修正既有頁、建立通路頁等 | 必填 |

## 批次流程

1. 每月或每次大型整理前，抓第三方榜單與官方統計，更新 [[30-研究/熱門信用卡候選清單|熱門信用卡候選清單]]。
2. 將 P0/P1 候選先查官方頁，不直接用第三方 headline 建推薦。
3. 若已有 wiki，檢查是否需要修復或補上最新活動。
4. 若無 wiki，先建立來源筆記，再依方法論升級卡片頁。
5. 若只是社群線索且無官方資料，放入 [[觀察清單]]。

## 並發但不分散判讀

適合並發：

- 抓多個榜單頁。
- 抓多張官方產品頁。
- 抽取卡名、活動期間、headline 與 URL。

不使用 sub-agent 直接完成：

- 判斷是否升級 wiki。
- 寫入來源筆記、卡片頁、策略頁。
- 修改公開/私有邊界相關檔案。

原因是這個 vault 需要一致的來源標準、命名規則、觀察清單與公開資料分層。
